Minimum Requirements for Project:

1. Your database must be relational, otherwise you will not receive any marks

2. Implement create, insert, retrieve/display, update, delete and search features

3. Must implement basic Registration and Login features

4. At least implement basic UI design(no need for css)

5. Must use php and html5

#  **1. CATEGORIES**

### **Strong / Weak:**

✔ **Strong entity**

### **Attributes:**

* category_id (simple, key)
* category_name (simple)

### **Primary Key:**

* **category_id**

### **Foreign Keys:**

* None (but referenced by Article)

### **Relationship Types:**

* **Categories — Article** : 1-to-Many

### **Cardinality:**

* One category → many articles
* One article → one category

### **Participation:**

* Category: **Partial** (a category can exist without an article)
* Article: **Total** (every article must have a category)

* Journalist — Login_Activity (1-to-Many)

### **Cardinality:**

### **Participation:**

* Login_Activity: Partial
* Actor entities: Partial
